| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

The APB peripheral address map assigns GPIO, UDMA/SPI, and SoC control regions that overlap. Since the APB node asserts a select for every range match, addresses in the overlap can target multiple peripherals and break intended peripheral isolation.

## 3. Expected RTL Evidence

| Evidence | 文件 | 行号 | 关键对象 | 评分用途 |
|---|---|---:|---|---|
| GPIO, UDMA/SPI, and SoC control ranges overlap | `third_party/hackatdac18/rtl/includes/periph_bus_defines.sv` | 24-34 | `GPIO_*`, `UDMA_*`, `SOC_CTRL_*` | Shows GPIO covers `0x1A10_1000-0x1A10_AFFF`, UDMA covers `0x1A10_2000-0x1A10_4FFF`, and SoC control covers `0x1A10_4000-0x1A10_4FFF`. |
| APB wrapper assigns those ranges to separate masters | `third_party/hackatdac18/ips/pulp_soc/rtl/pulp_soc/periph_bus_wrap.sv` | 57-67 | `gpio_master`, `udma_master`, `soc_ctrl_master` | Confirms the overlapping ranges are used for distinct APB targets. |
| APB decoder does not enforce unique selection | `third_party/hackatdac18/ips/apb/apb_node/apb_node.sv` | 59-64 | `psel_o[i]` | Shows each matching target independently receives `psel_o[i]`. |
| GPIO connected to APB bus | `third_party/hackatdac18/ips/pulp_soc/rtl/pulp_soc/soc_peripherals.sv` | 352-365 | `s_gpio_bus` | Confirms GPIO is an APB target. |
| UDMA/SPI and SoC control connected to APB bus | `third_party/hackatdac18/ips/pulp_soc/rtl/pulp_soc/soc_peripherals.sv` | 384-421, 495-510 | `s_udma_bus`, `s_soc_ctrl_bus` | Confirms SPI/UDMA and SoC control are APB targets. |

Minimum sufficient evidence:

```text
periph_bus_defines.sv:25-34
GPIO:    0x1A10_1000 - 0x1A10_AFFF
UDMA:    0x1A10_2000 - 0x1A10_4FFF

```

## 4. Hit Criteria

A finding is a hit if it satisfies all of the following:

- Identifies the GPIO, SPI/UDMA, and SoC control APB address-range overlap.
- Locates `periph_bus_defines.sv` or equivalent range definitions.
- Explains why overlapping ranges break peripheral isolation under the APB decoder.
- Cites `apb_node.sv` or otherwise explains independent target selection.

## 5. Partial Hit Criteria

- Identifies only one overlap pair, such as GPIO/UDMA or UDMA/SoC control.
- Locates the range definitions but does not explain the decoder behavior.
- Reports generic peripheral aliasing in the correct APB files.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports GPIO register lock issues instead of address-range overlap.
- Claims a GPIO/SPI data-path vulnerability without address-map evidence.
- Uses H18-001 evidence but fails to mention GPIO.

False negative condition:

- If the input scope contains `periph_bus_defines.sv`, `periph_bus_wrap.sv`, and `apb_node.sv`, and the agent does not report the GPIO/SPI/SoC-control overlap, mark H18-008 as false negative.

## 7. Evaluation Notes

- This case should be distinguished from H18-001 by requiring GPIO to be part of the reported overlap.
- Core evidence is the three-way APB address interval conflict.
