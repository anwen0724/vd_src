## 2. Expected Bug Semantics

The APB peripheral address map gives the UDMA/SPI master region and the SoC control region an overlapping address interval. The APB node selects every slave whose range matches `paddr_i`; therefore an access in the overlap can assert multiple peripheral selects instead of routing uniquely to one resource.

## 4. Hit Criteria

A finding is a hit if it satisfies all of the following:

- Identifies the address-range overlap between UDMA/SPI master and SoC control.
- Locates `periph_bus_defines.sv`, `periph_bus_wrap.sv`, or equivalent APB address-map code.
- Mentions the overlapping interval around `0x1A10_4000-0x1A10_4FFF`.
- Explains that the APB decoder can select more than one target because `psel_o[i]` is generated independently for each matching range.

## 5. Partial Hit Criteria

- Identifies a peripheral address overlap but does not tie it specifically to SPI/UDMA and SoC control.
- Locates only the address definitions but not the APB selection logic.
- Reports a possible APB aliasing issue in the correct files without precise range reasoning.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims a generic SPI vulnerability without address-map evidence.
- Reports an overlap involving unrelated peripherals or nonexistent ranges.
- Treats a documented adjacent range as an overlap without proving intersecting intervals.

False negative condition:

- If the input scope contains `periph_bus_defines.sv`, `periph_bus_wrap.sv`, and `apb_node.sv`, and the agent does not report the UDMA/SPI-to-SoC-control address overlap, mark H18-001 as false negative.

## 7. Evaluation Notes

- This case overlaps with H18-008, but H18-001 should be scored specifically on the SPI/UDMA master versus SoC control range.
- Core evidence is the address interval conflict plus APB independent `psel_o[i]` generation.
