## 2. Expected Bug Semantics

The SoC interconnect forwards a broad peripheral interval into the APB bridge, and the downstream APB node also contains overlapping peripheral ranges. Because the APB node generates target selects from inclusive address comparisons independently for every slave, an address can alias across APB targets instead of mapping to exactly one peripheral.

## 4. Hit Criteria

A finding is a hit if it satisfies all of the following:

- Identifies APB/peripheral address aliasing or incorrect APB range decoding.
- Cites the APB/interconnect range definitions or APB node selection logic.
- Explains that inclusive, independent comparisons can select multiple targets or route an address into an unintended APB resource.
- Distinguishes the issue from a single-register access-control bug.

## 5. Partial Hit Criteria

- Finds overlapping APB ranges but does not connect them to aliasing behavior.
- Finds the broad interconnect APB range but not the downstream APB node behavior.
- Describes generic APB decode risk with correct file references but incomplete interval reasoning.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports a non-APB memory aliasing issue without APB range evidence.
- Reports address aliasing in unrelated bus code.
- Claims a fixed-priority decoder without evidence.

False negative condition:

- If the input scope contains the SoC interconnect, APB wrapper, APB node, and APB range definitions, and the agent does not report an APB address aliasing/range issue, mark H18-006 as false negative.

## 7. Evaluation Notes

- This case is broader than H18-001/H18-008. H18-006 should be scored on APB aliasing as a bus-level address-range problem, not only on one named peripheral pair.
