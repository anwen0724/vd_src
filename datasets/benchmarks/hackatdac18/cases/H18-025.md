## 2. Expected Bug Semantics

CSR instructions can drive the CSR access/write path for privileged CSRs without a privilege-level mediation check in the `zero-riscy` decode path. The decoder exposes CSR operations from `OPCODE_SYSTEM`, and the CSR register file updates privileged machine CSRs such as `mstatus`, `mepc`, and `mcause` based on `csr_op_i` and `csr_addr_i`.

## 4. Hit Criteria

A finding is a hit if it satisfies all of the following:

- Identifies that privileged CSR writes lack privilege-level mediation.
- Locates the `zero-riscy` CSR decode/write path.
- Mentions `csr_access_o`, `csr_op`, `csr_illegal`, `csr_addr_i`, or equivalent CSR control signals.
- Cites privileged CSR targets such as `mstatus` (`12'h300`), `mepc` (`12'h341`), or `mcause` (`12'h342`).
- Explains that CSR operation validity is checked, but current privilege level is not used to block lower-privilege CSR writes.

## 5. Partial Hit Criteria

A finding can be a partial hit if it satisfies one of the following:

- Identifies a CSR privilege risk but does not locate the `zero-riscy` decoder.
- Locates the CSR write path but does not connect it to privileged CSR targets.
- Reports a `needs_more_evidence` warning that correctly targets missing privilege checks on CSR access.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports generic CSR access without mentioning privilege or privileged CSR targets.
- Uses evidence only from `ips/riscv/riscv_decoder.sv`; that is a different implementation and includes a current privilege check.
- Claims a debug CSR path issue only, without connecting to user instruction CSR decode.

False negative condition:

- If the input scope contains `zero-riscy` CSR decode and CSR register files, and the agent does not report missing privilege mediation for privileged CSR writes, mark H18-025 as false negative.

## 7. Evaluation Notes

- Optional semantic check: source = unprivileged/user CSR instruction; mediation = missing current privilege check; sink = privileged machine CSRs.
- Core evidence is `zeroriscy_decoder.sv:508-536` plus `zeroriscy_cs_registers.sv:188-209`.
- Be careful not to score evidence from the separate `ips/riscv` decoder as a hit for this case.
