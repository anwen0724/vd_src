## 2. Expected Bug Semantics

The external JTAG path can be routed into the SoC debug/LINT access path without a password or authentication guard on that path. This exposes an external requester path to internal debug/bus resources through TAP selection and `lint_jtag_wrap`.

## 4. Hit Criteria

A finding is a hit if it satisfies all of the following:

- Identifies that a JTAG path lacks password/authentication protection.
- Locates the unprotected path through `jtag_tap_top`, `pulp_soc.sv`, `lint_jtag_wrap.sv`, or `adbg_lintonly_top.sv`.
- Shows that external JTAG-derived signals reach an internal debug/LINT bus path.
- Notes the absence of a password/auth guard on this JTAG-to-LINT path.
- Distinguishes this unprotected path from the separate advanced debug TAP password logic used by H18-009/H18-010/H18-012.

## 5. Partial Hit Criteria

A finding can be a partial hit if it satisfies one of the following:

- Identifies an exposed JTAG debug risk but does not trace the internal LINT/debug bus sink.
- Locates `lint_jtag_wrap` or `jtag_tap_top` but does not explicitly discuss missing authentication.
- Reports a `needs_more_evidence` warning that correctly targets JTAG-to-internal-bus authorization.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports the flawed password logic in `adbg_tap_top.v` only; that belongs to H18-009/H18-010/H18-012.
- Claims all JTAG paths are password-protected because another advanced debug TAP has password-checking code.
- Reports generic JTAG exposure without showing a path to internal debug/bus resources.

False negative condition:

- If the input scope contains the JTAG TAP, SoC JTAG connections, and `lint_jtag_wrap.sv`, and the agent does not report the unprotected JTAG-to-internal debug/bus path, mark H18-028 as false negative.
