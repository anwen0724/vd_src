## 2. Expected Bug Semantics

The interrupt controller exposes APB-writable interrupt registers, but the write paths for interrupt mask, interrupt pending, and acknowledge state do not require `core_secure_mode_i`. As a result, the secure-mode signal is present at the module boundary but is not used to mediate writes to interrupt control registers.

## 4. Hit Criteria

A finding is a hit if it satisfies all of the following:

- Identifies that interrupt register writes are not gated by secure mode.
- Locates `apb_interrupt_cntrl.sv` or the `apb_interrupt_cntrl` module.
- Mentions `core_secure_mode_i` and one or more interrupt register states: `r_mask`, `r_int`, or `r_ack`.
- Cites the APB write path or the next-state update logic using `apb_slave.pwdata`.
- Explains that secure-mode input is absent from the write conditions.

## 5. Partial Hit Criteria

A finding can be a partial hit if it satisfies one of the following:

- Identifies unprotected interrupt register writes but does not mention `core_secure_mode_i`.
- Locates `apb_interrupt_cntrl.sv` write paths but does not explain the missing secure-mode mediation.
- Reports a `potential_warning` whose evidence targets secure-mode/register-write mismatch.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports generic APB writability without tying it to interrupt registers or secure mode.
- Reports an interrupt functional bug without discussing write authorization.
- Claims secure-mode gating exists based only on the presence of `core_secure_mode_i`.

False negative condition:

- If the input scope contains `apb_interrupt_cntrl.sv` and the FC subsystem instance, and the agent does not report missing secure-mode gating on interrupt register writes, mark H18-027 as false negative.
