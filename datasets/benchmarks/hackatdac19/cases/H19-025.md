## 1. Case Identity

| 字段 | 内容 |

| `case_id` | H19-025 |

| `official_description` | SATP register (write) accessible in Supervisor mode even if TVM is enabled |

| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

The CSR write path updates `satp_d` when `CSR_SATP` is written and the mode is supported. Although TVM state exists in `mstatus_q.tvm`, the SATP write branch does not gate or trap writes based on TVM.

## 4. Hit Criteria

- Identifies SATP write access is not mediated by TVM.
- Cites the `CSR_SATP` write branch and absence of a TVM guard.
- Explains that the condition is specific to Supervisor mode with TVM enabled.

## 5. Partial Hit Criteria

- Reports SATP writeability but does not mention TVM.
- Finds TVM state but does not connect it to SATP write mediation.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims no SATP write path exists.
- Confuses write access with read access.

False negative condition:

- If the input scope contains `csr_regfile.sv`, but the agent does not report missing TVM mediation for SATP writes, mark H19-025 as false negative.

## 7. Evaluation Notes

This case is distinct from H19-024, which concerns SATP reads.
