## 2. Expected Bug Semantics

The JTAG authentication key is derived from a fixed ROM2 constant. `rom2.sv` initializes a constant word intended for JTAG, `ariane_peripherals.sv` assigns `jtag_key` from `key_reg_out[1]`, and `dmi_jtag.sv` copies this input into the internal `pass` register on reset. This makes the JTAG key RTL-visible and hardcoded.

| JTAG module copies key into password register | `third_party/hackatdac19/src/debug/dmi_jtag.sv` | 228-233 | `pass`, `jtag_key` | Shows the key is used as the JTAG password reference. |

## 4. Hit Criteria

- Identifies that the JTAG key/password reference is hardcoded in RTL-visible ROM2 constants.
- Cites the ROM2 constant and the path into `jtag_key` / `pass`.
- Explains why this is a hardcoded-secret issue.

## 5. Partial Hit Criteria

- Finds a ROM2 constant but does not connect it to JTAG authentication.
- Reports generic hardcoded secret without identifying `jtag_key`.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims AES key readback only; that is H19-010.
- Claims secure registers are AXI-exposed only; that is H19-045.

False negative condition:

- If the input scope contains `rom2.sv`, `ariane_peripherals.sv`, and `dmi_jtag.sv`, but the agent does not report the hardcoded JTAG key path, mark H19-054 as false negative.

## 7. Evaluation Notes

This case is about the JTAG authentication secret source, not about general ROM2 register exposure.
