| `official_description` | Secure registers are accessible via AXI |

## 2. Expected Bug Semantics

The ROM2/fuse block contains secure key/configuration registers and exposes them through a normal AXI-to-memory path. The `rom2` module returns `secure_reg[raddr_q]` on reads and also allows writes when `we_i` is asserted. This makes secure registers reachable through the AXI-accessible ROM2 path.

## 4. Hit Criteria

- Identifies that secure ROM2/fuse registers are exposed through an AXI path.
- Cites both the SoC connection and the `rom2.sv` secure register read/write behavior.
- Explains why this is a protected-resource exposure.

## 5. Partial Hit Criteria

- Finds `secure_reg` but does not connect it to the AXI path.
- Reports generic key storage without identifying ROM2/fuse exposure.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims AES wrapper key readback only; that is H19-010.
- Claims boot ROM writeability; that is H19-017.

False negative condition:

- If the input scope contains `rom2.sv`, `ariane_peripherals.sv`, and `ariane_testharness.sv`, but the agent does not report AXI exposure of secure registers, mark H19-045 as false negative.

## 7. Evaluation Notes

This case should be scored on secure-register exposure through AXI/ROM2, not on AES-specific key readback.
