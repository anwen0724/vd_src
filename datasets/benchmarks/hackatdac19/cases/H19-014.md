## 2. Expected Bug Semantics

The AES peripheral is mapped as an AXI/APB-accessible peripheral and exposes a register interface that accepts plaintext/start writes and returns ciphertext/status. The SoC access-control configuration permits AES connectivity for lower privilege states through the AXI connectivity map. This makes the AES operation interface available as a crypto oracle.

## 4. Hit Criteria

- Identifies that the AES encrypt/decrypt interface is exposed as a crypto oracle.
- Cites the AES register interface and the SoC access path.
- Explains why lower-privilege access to this interface is a protected-operation exposure.

## 5. Partial Hit Criteria

- Finds AES key readback only; that is H19-010.
- Finds AES register access but does not explain oracle-style operation exposure.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims AES key is hardcoded only; that is H19-015.
- Claims AES intermediate state leakage only; that is H19-011.

False negative condition:

- If the input scope contains the AES wrapper and SoC access-control path, but the agent does not report AES oracle exposure, mark H19-014 as false negative.

## 7. Evaluation Notes

This case should be scored on operation exposure, not key readback or intermediate-state readback.
