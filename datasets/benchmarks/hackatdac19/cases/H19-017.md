## 2. Expected Bug Semantics

The boot ROM instance is connected behind an AXI-to-memory adapter that provides both request and write-enable signals. Inside `bootrom.sv`, when `req_i` is asserted and `we_i` is high, the implementation writes `wdata_i` into `mem[...]`. This makes the boot ROM writable through the bus path instead of read-only.

## 4. Hit Criteria

- Identifies boot ROM is writable through the bus path.
- Cites both the `axi2mem`/`bootrom` connection and the `mem[...] <= wdata_i` write branch.
- Explains why this violates boot ROM write protection.

## 5. Partial Hit Criteria

- Finds the boot ROM write branch but not the bus connection.
- Reports boot ROM corruption without line-level RTL evidence.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims firmware update support without evidence.
- Reports ROM2/fuse writeability instead of boot ROM writeability.

False negative condition:

- If the input scope contains `bootrom.sv` and `ariane_testharness.sv`, but the agent does not report the boot ROM write path, mark H19-017 as false negative.

## 7. Evaluation Notes

This case is confirmed by the vulnerable RTL and official fix branch.
