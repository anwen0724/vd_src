## 2. Expected Bug Semantics

The JTAG password flag `pass_chk` is used to control access-sensitive behavior, including the JTAG-derived `umode_o` signal. However, the reset block in `dmi_jtag.sv` resets several state registers and does not assign `pass_chk` back to an unauthenticated value. This can leave the password-check state improperly reset.
