## 2. Expected Bug Semantics

The debug CSR path allows DMI writes to update `dmcontrol`. The `ndmreset` bit from `dmcontrol_q` is exported as `ndmreset_o`, and the test harness uses this signal to generate `ndmreset_n`, which resets multiple SoC components. This exposes a secondary reset control through the debug control path.

## 4. Hit Criteria

- Identifies that debug/DMI control can drive a secondary reset signal.
- Cites `dmcontrol_q.ndmreset` to `ndmreset_o` and the test harness reset generation.
- Explains why this is a reset-control exposure.

## 5. Partial Hit Criteria

- Finds `ndmreset_o` but does not connect it to DMI `dmcontrol` writes.
- Reports generic debug exposure without identifying reset control.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims JTAG password flag reset residue; that is H19-051.
- Claims debug is simply always authenticated; that is H19-005.

False negative condition:

- If the input scope contains `dm_csrs.sv`, `dm_top.sv`, and `ariane_testharness.sv`, but the agent does not report debug-controlled secondary reset exposure, mark H19-056 as false negative.

## 7. Evaluation Notes

This case should be scored on the reset-control path from debug CSR state to SoC reset distribution.
