## 2. Expected Bug Semantics

The AES wrapper exposes internal intermediate state through `external_bus_io.rdata`. The AES core outputs `inter_state`, and the wrapper maps `inter_state[127:0]` to read addresses 22-25. This leaks internal encryption state through the external bus interface.

## 4. Hit Criteria

- Identifies that AES intermediate/internal state is readable externally.
- Cites `inter_state` readback in `aes_wrapper.sv`.
- Distinguishes this from the AES key readback case.

## 5. Partial Hit Criteria

- Reports generic AES register exposure but does not identify `inter_state`.
- Finds `inter_state` but does not connect it to the external read interface.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports only ciphertext output as internal-state exposure.
- Merges this case entirely with H19-010 without identifying `inter_state`.

False negative condition:

- If the input scope contains `aes_wrapper.sv`, but the agent does not report the `inter_state` readback path, mark H19-011 as false negative.

## 7. Evaluation Notes

Score this as internal state exposure. Key exposure alone is not sufficient for a full hit on this case.
