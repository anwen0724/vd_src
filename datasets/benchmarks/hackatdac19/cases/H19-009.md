## 2. Expected Bug Semantics

The CSR register file outputs machine privilege when either debug mode is active or `umode_i` is asserted. This means a user-mode control input can force `priv_lvl_o` to `PRIV_LVL_M`, allowing machine-level execution semantics outside the intended privilege state.

## 3. Expected RTL Evidence

| Evidence | 文件 | 行号 | 关键对象 | 评分用途 |
|---|---|---:|---|---|
| Top-level user-mode input | `third_party/hackatdac19/src/ariane.sv` | 50-51, 496-520 | `umode_i`, `csr_regfile` | Shows `umode_i` is connected into the CSR register file. |
| Privilege output assignment | `third_party/hackatdac19/src/csr_regfile.sv` | 937-938 | `priv_lvl_o`, `debug_mode_q`, `umode_i` | Shows `umode_i` forces machine privilege. |
| Official fix reference | `origin/fix_cwe_1244_in_csr_regfile` | diff | `priv_lvl_o` assignment | Fix removes `umode_i` from the condition that grants machine privilege. |

## 4. Hit Criteria

- Identifies that `umode_i` can force `priv_lvl_o` to `PRIV_LVL_M`.
- Cites `csr_regfile.sv` line 938 or equivalent.
- Explains why this can allow user-mode execution to be treated as machine mode.

## 5. Partial Hit Criteria

- Reports a privilege escalation in CSR logic but does not identify `umode_i`.
- Finds the official fix branch but does not explain the vulnerable condition.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims arbitrary CSR access without tying it to `priv_lvl_o`.
- Treats normal debug-mode machine privilege as the bug.

False negative condition:

- If the input scope contains `csr_regfile.sv` and the agent does not report the `umode_i` privilege escalation condition, mark H19-009 as false negative.

## 7. Evaluation Notes

This case is distinct from H19-020, H19-024, and H19-025. It concerns how the current privilege level is produced, not a specific CSR read/write rule.
