# H19-058 Case GT

## 1. Case Identity

| 字段 | 内容 |
|---|---|
| `case_id` | H19-058 |
| `official_id` | #58 |
| `official_description` | Register locks can be reprogrammed after boot-up |
| `task_category` | `register_lock/lifecycle` |
| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

ROM2 holds security configuration values, including access-control configuration words that are exported as `access_ctrl_reg`. Although these values are initialized from constants at reset, `rom2.sv` also allows writes to `secure_reg` when `we_i` is asserted. As a result, lock or access-control configuration state can be reprogrammed after boot through the ROM2 register path.

## 4. Hit Criteria

- Identifies that security/access-control configuration registers can be reprogrammed after boot.
- Cites both the ROM2 write path and the `access_ctrl_reg` derivation.
- Explains this as a lifecycle or register-lock failure, not only generic ROM2 exposure.

## 5. Partial Hit Criteria

- Finds ROM2 secure registers are writable but does not connect them to access-control configuration.
- Reports AXI exposure of secure registers only; that is H19-045.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims boot ROM memory writeability; that is H19-017.
- Claims hardcoded JTAG key only; that is H19-054.

False negative condition:

- If the input scope contains `rom2.sv`, `ariane_peripherals.sv`, and `ariane_testharness.sv`, but the agent does not report post-boot reprogrammability of access-control or lock-like security configuration, mark H19-058 as false negative.

## 7. Evaluation Notes

This case should be scored on mutable security configuration after boot. It is related to H19-045 but has a different expected conclusion.
