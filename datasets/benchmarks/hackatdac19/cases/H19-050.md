## 2. Expected Bug Semantics

The debug module includes a System Bus Access path that can issue AXI master requests. The SoC harness connects the debug AXI master to the same fabric that includes ROM2/FUSE memory. Since ROM2/FUSE contains key and security-configuration registers, authorized debug access can reach FUSE memory through the system bus path.

## 4. Hit Criteria

- Identifies that authorized debug/SBA can access ROM2/FUSE memory.
- Cites the debug SBA AXI master path and the ROM2/FUSE mapping.
- Explains why this is overbroad debug access to protected resource state.

## 5. Partial Hit Criteria

- Finds ROM2/FUSE exposure but not through debug SBA.
- Reports debug authentication weakness only; that is H19-005.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims ROM2 is exposed through ordinary AXI only; that is H19-045.
- Claims JTAG password issues without the debug SBA path.

False negative condition:

- If the input scope contains `dm_top.sv`, `dm_csrs.sv`, `ariane_testharness.sv`, and `rom2.sv`, but the agent does not report debug access to ROM2/FUSE memory, mark H19-050 as false negative.

## 7. Evaluation Notes

This case is related to ROM2 exposure, but the expected finding must mention the debug System Bus Access path.
