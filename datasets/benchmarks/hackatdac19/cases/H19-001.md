## 2. Expected Bug Semantics

The AXI connectivity mapping grants PLIC connectivity when either the PLIC access-control bit is enabled or the CLINT access-control bit is enabled. Since `ariane_soc::PLIC` is index 6 and `ariane_soc::CLINT` is index 7, the hard-coded condition `((j==6) && access_ctrl_i[i][7][priv_lvl_i])` lets CLINT authorization imply PLIC authorization.

## 4. Hit Criteria

- Identifies that PLIC access can be granted through CLINT authorization.
- Cites `axi_node_intf_wrap.sv` line 430 or equivalent connectivity mapping.
- Explains the index relation `PLIC = 6` and `CLINT = 7`.
- Explains this is an access-control mediation failure in the AXI connectivity map.

## 5. Partial Hit Criteria

- Reports a generic AXI access-control bypass without naming CLINT/PLIC.
- Finds `access_ctrl_i` but does not explain the hard-coded cross-peripheral condition.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims an address-range overlap without evidence of the connectivity-map bypass.
- Reports a PLIC bug unrelated to CLINT authorization.

False negative condition:

- If the input scope contains `axi_node_intf_wrap.sv` and `ariane_soc_pkg.sv`, but the agent does not report the CLINT-to-PLIC authorization bypass, mark H19-001 as false negative.

## 7. Evaluation Notes

This case is confirmed by both vulnerable RTL and the official fix branch. Score it as an access-control connectivity failure, not as a generic interrupt-controller issue.
