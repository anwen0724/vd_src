# H19-047 Case GT

## 1. Case Identity

| 字段 | 内容 |
|---|---|
| `case_id` | H19-047 |
| `official_id` | #47 |
| `official_description` | Some JTAG instructions can be accessed without password |
| `task_category` | `JTAG/authorization_bypass` |
| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

The JTAG DMI access path only checks `pass_chk` for DMI reads, while DMI writes can enter the `Write` state without the password flag being set. This means at least part of the JTAG/DMI operation surface is not consistently mediated by the password check.

| DMIACCESS selection | `third_party/hackatdac19/src/debug/dmi_jtag_tap.sv` | 175-187 | `dmi_access_o` | Shows DMI access can be selected by the TAP instruction decoder. |

| DMI request generation | `third_party/hackatdac19/src/debug/dmi_jtag.sv` | 85-90 | `dmi_req.op`, `dmi_req.addr`, `dmi_req.data` | Shows selected JTAG state becomes a DMI request. |

## 4. Hit Criteria

- Identifies that DMI/JTAG operations are not consistently password guarded.
- Cites the `DTM_WRITE` transition in `dmi_jtag.sv` that lacks a `pass_chk` condition.
- Connects this to the JTAG TAP `DMIACCESS` path.

## 5. Partial Hit Criteria

- Reports a generic JTAG password issue without explaining the read/write guard asymmetry.
- Finds `DMIACCESS` but does not connect it to missing password mediation.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims the JTAG key is hardcoded only; that is H19-054.
- Claims unlocking JTAG elevates processor privilege only; that is H19-048.

False negative condition:

- If the input scope contains `dmi_jtag.sv` and `dmi_jtag_tap.sv`, but the agent does not report password-bypassable JTAG/DMI operations, mark H19-047 as false negative.

## 7. Evaluation Notes

This case is about inconsistent password mediation across JTAG/DMI operations.
