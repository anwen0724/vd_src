## 2. Expected Bug Semantics

DRAM is a mapped SoC target and its connectivity is controlled by the same privilege-indexed access-control matrix used by the AXI node wrapper. The ROM2 access-control constants enable DRAM connectivity for lower privilege bits, so lower-privilege requesters can access the full DRAM region through the SoC fabric.

## 4. Hit Criteria

- Identifies lower-privilege access to the DRAM region.
- Cites the DRAM mapping and the privilege-indexed access-control path.
- Explains why this is a memory-protection/access-control failure.

## 5. Partial Hit Criteria

- Finds DRAM address mapping but does not connect it to privilege/access-control bits.
- Reports generic address-map exposure without identifying lower-privilege access.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims boot ROM writeability; that is H19-017.
- Claims ROM2 secure-register exposure; that is H19-045.

False negative condition:

- If the input scope contains the DRAM mapping and AXI access-control path, but the agent does not report lower-privilege DRAM access, mark H19-041 as false negative.

## 7. Evaluation Notes

This case should be scored as DRAM memory-access policy failure through the SoC fabric.
