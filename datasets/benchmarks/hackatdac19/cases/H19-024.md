## 2. Expected Bug Semantics

The CSR register file exposes `CSR_SATP` reads through the normal supervisor CSR read path. Although `mstatus_q.tvm` is exported as `tvm_o`, the CSR read logic and privilege exception logic do not apply a TVM-specific trap for SATP reads.

## 3. Expected RTL Evidence

| Evidence | 文件 | 行号 | 关键对象 | 评分用途 |
|---|---|---:|---|---|
| SATP CSR definition | `third_party/hackatdac19/include/riscv_pkg.sv` | 345-355 | `CSR_SATP` | Shows SATP is defined as a supervisor CSR. |
| SATP read path | `third_party/hackatdac19/src/csr_regfile.sv` | 202-209 | `CSR_SATP`, `csr_rdata = satp_q` | Shows SATP is returned on reads. |
| Generic privilege check only | `third_party/hackatdac19/src/csr_regfile.sv` | 848-856 | `csr_exception_o` | Shows CSR exception logic checks privilege level but no TVM-specific SATP read rule. |
| TVM state exists | `third_party/hackatdac19/src/csr_regfile.sv` | 944-952 | `tvm_o`, `mstatus_q.tvm` | Shows TVM state exists but is not used in the SATP read path. |

## 4. Hit Criteria

- Identifies SATP read access is not mediated by TVM.
- Cites `csr_rdata = satp_q` and absence of TVM-specific check in the relevant CSR exception path.
- Explains that the condition is specific to Supervisor mode with TVM enabled.

## 5. Partial Hit Criteria

- Reports generic SATP accessibility but does not mention TVM.
- Finds TVM state but does not connect it to SATP read mediation.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims SATP is always illegal to read.
- Confuses read access with write access.

False negative condition:

- If the input scope contains `csr_regfile.sv`, but the agent does not report missing TVM mediation for SATP reads, mark H19-024 as false negative.

## 7. Evaluation Notes

This case is distinct from H19-025, which concerns SATP writes.
