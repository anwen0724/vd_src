## 2. Expected Bug Semantics

The AES wrapper receives a 192-bit key and assigns it to `key_big`. The APB-readable register interface returns the key words at addresses 16-21 without an authorization guard. A requester reaching the AES bus interface can therefore read key material through normal register reads.

## 4. Hit Criteria

- Identifies that AES key material is readable through the AES register interface.
- Cites the `key_big` readback cases in `aes_wrapper.sv`.
- Explains that the readback path lacks an access-control guard.

## 5. Partial Hit Criteria

- Reports AES key exposure but does not locate the readback addresses.
- Finds `key_big` but does not connect it to `external_bus_io.rdata`.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims the key is writable without evidence.
- Reports a cryptographic algorithm weakness rather than key exposure.

False negative condition:

- If the input scope contains `aes_wrapper.sv`, but the agent does not report the key readback path, mark H19-010 as false negative.

## 7. Evaluation Notes

This case is scored on key exposure. H19-011 is scored separately on intermediate AES state exposure.
