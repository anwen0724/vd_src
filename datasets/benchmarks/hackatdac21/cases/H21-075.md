# H21-075 Case GT

## 1. Case Identity

| 字段 | 内容 |
|---|---|
| `case_id` | H21-075 |
| `official_id` | #75 |
| `official_description` | HMAC key registers are not locked by register locks |
| `task_category` | `hmac/key_register_lock` |
| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

HMAC key and key-derived registers are written under `reglk_ctrl_i[5]`, while key-hash readbacks are guarded by `reglk_ctrl_i[4]`. The same security-sensitive key material is therefore not protected by a consistent lock domain.

## 4. Hit Criteria

- Reports inconsistent register-lock protection for HMAC key/key-derived registers.
- Cites write guard `reglk_ctrl_i[5]` and read guard `reglk_ctrl_i[4]`.
- Explains why this weakens key register protection.

## 5. Partial Hit Criteria

- Reports HMAC user-mode access only.
- Reports HMAC key registers but does not compare lock bits.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims HMAC key material is unconditionally readable without evidence.
- Reports HMAC reset residue only.

False negative condition:

- If the input scope contains `hmac_wrapper.sv`, but the agent does not report inconsistent lock mediation over HMAC key-derived registers, mark H21-075 as false negative.

## 7. Evaluation Notes

This case is a register-lock consistency issue over sensitive HMAC state.
