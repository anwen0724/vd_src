## 2. Expected Bug Semantics

The SoC-level chicken-bit input `we_flag_4` is wired into `dmi_jtag` as `we_flag` and participates in JTAG authentication/read mediation, while the expected JTAG hash still comes from fuse memory and is compared against the computed password hash.

## 4. Hit Criteria

- Identifies that a chicken-bit-like `we_flag_4` reaches JTAG authentication/read mediation.
- Connects `we_flag` to the read authorization condition and expected-hash authentication path.
- Explains why this can corrupt or bypass the intended JTAG password mediation.

## 5. Partial Hit Criteria

- Finds the expected JTAG hash comparison but misses `we_flag_4`.
- Finds `we_flag` but does not connect it to JTAG authorization behavior.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports a generic JTAG issue without citing `we_flag_4`/`we_flag` or the expected-hash path.
- Claims the expected hash value itself is modified without evidence.

False negative condition:

- If the input scope contains the listed evidence files but the agent does not report this supported vulnerability, mark this case as false negative.

## 7. Evaluation Notes

This case was reviewed on 2026-06-22. The evidence was refined to focus on the source-supported chicken-bit influence on JTAG authentication/read mediation rather than overclaiming a direct rewrite of `exp_hash`.
