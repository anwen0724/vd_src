# H21-060 Case GT

## 1. Case Identity

| 字段 | 内容 |
|---|---|
| `case_id` | H21-060 |
| `official_id` | #60 |
| `official_description` | AES leaks secret key0 because default case does not cover all possible values |
| `task_category` | `aes/default_read_key_leak` |
| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

AES0 read logic preloads `rdata` from `key0[address[8:3]]` before the address case statement. The case/default logic does not always overwrite that value, so key0 can be returned through incomplete default handling.

## 4. Hit Criteria

- Reports AES0 key0 leakage through default/incomplete read handling.
- Cites `rdata = key0[address[8:3]]`.
- Distinguishes this from generic AES state exposure.

## 5. Partial Hit Criteria

- Reports AES key exposure but not the default read mechanism.
- Finds key registers but not the faulty read pre-assignment.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims plaintext residue only.
- Reports AES2 exposure without AES0 key0/default-read evidence.

False negative condition:

- If the input scope contains `aes0_wrapper.sv`, but the agent does not report key0 leakage through default read handling, mark H21-060 as false negative.

## 7. Evaluation Notes

This case is close to H21-013 but has a more specific mechanism: AES0 key0 leakage through default read behavior.
