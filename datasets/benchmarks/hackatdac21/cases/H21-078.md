## 1. Case Identity

| 字段 | 内容 |

| `case_id` | H21-078 |

| `official_description` | Some PKT registers are not locked by register locks |

| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

The PKT wrapper has `reglk_ctrl_i` and exposes fuse-related control registers. The write map lets software update `fuse_req_o` and `fuse_addr_o` without register-lock checks, while only selected readbacks use `reglk_ctrl_i`.

## 4. Hit Criteria

- Reports that some PKT fuse-control registers are not register-lock protected.
- Cites direct writes to `fuse_req_o` and `fuse_addr_o`.
- Explains inconsistency with lock-protected PKT read paths.

## 5. Partial Hit Criteria

- Reports PKT fuse leakage but not register-lock omission.
- Finds `reglk_ctrl_i` but does not identify the unguarded write map.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims all PKT registers are unprotected without evidence.
- Reports HMAC-to-PKT access coupling only.

False negative condition:

- If the input scope contains `pkt_wrapper.sv`, but the agent does not report unguarded PKT fuse-control register writes, mark H21-078 as false negative.

## 7. Evaluation Notes

This is a register-lock case over PKT control registers, distinct from H21-059 fuse data exposure.
