# H21-036 Case GT

## 1. Case Identity

| 字段 | 内容 |
|---|---|
| `case_id` | H21-036 |
| `official_id` | #36 |
| `official_description` | SHA input data not cleared after HASH computation |
| `task_category` | `sha/state_residue` |
| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

SHA input words are stored in `data[0]` through `data[15]`. The same words remain readable through the register map, and the normal hash-control/read path does not show a post-computation clear of those input registers.

## 4. Hit Criteria

- Reports residual SHA input data after hash computation.
- Cites both the write map and read map for `data[0]` through `data[15]`.
- Explains this as sensitive state residue rather than a hash algorithm flaw.

## 5. Partial Hit Criteria

- Finds readable SHA data registers but does not state the post-computation residue problem.
- Reports generic SHA register exposure without tying to input data persistence.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims SHA digest exposure as the bug instead of input data residue.
- Reports missing register locks without identifying retained input data.

False negative condition:

- If the input scope contains `sha256_wrapper.sv`, but the agent does not report readable uncleared SHA input data, mark H21-036 as false negative.

## 7. Evaluation Notes

The evidence is source-level: writable input registers remain readable; no normal completion clear is visible in the wrapper logic.
