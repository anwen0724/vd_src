## 2. Expected Bug Semantics

AES2 access is gated by `acc_ctrl_c[priv_lvl_i][11]`. Because access-control entries reset to all ones, the privilege-indexed matrix grants AES2 access by default unless later restricted, making AES2 protected registers accessible from user mode.

## 4. Hit Criteria

- Reports user-mode accessibility of AES2 protected registers.
- Connects AES2 access gate to `acc_ctrl_c[priv_lvl_i][11]`.
- Explains why reset-open access control enables the privilege issue.

## 5. Partial Hit Criteria

- Reports AES2 register exposure but omits user-mode/access-control reasoning.
- Reports global access-control reset but does not identify AES2.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims AES0 plaintext residue only.
- Claims AES algorithm weakness unrelated to privilege gating.

False negative condition:

- If the input scope contains `riscv_peripherals.sv`, `acct_wrapper.sv`, and `aes2_wrapper.sv`, but the agent does not report user-mode accessibility of AES2 protected registers, mark H21-073 as false negative.
