# H21-039 Case GT

## 1. Case Identity

| 字段 | 内容 |
|---|---|
| `case_id` | H21-039 |
| `official_id` | #39 |
| `official_description` | AES plain text is left uncleared after the encryption is over in the peripheral registers |
| `task_category` | `aes/state_residue` |
| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

AES plaintext words are stored in `p_c[0]` through `p_c[3]`. They are reset only on reset, written through the peripheral register map, and remain readable through the same register interface after encryption unless overwritten or reset.

## 4. Hit Criteria

- Reports AES plaintext residue in memory-mapped peripheral registers.
- Cites both plaintext write and read paths for `p_c`.
- States that normal encryption completion does not clear or hide the plaintext registers.

## 5. Partial Hit Criteria

- Reports AES register exposure but does not distinguish plaintext residue.
- Identifies `p_c` registers but not their continued readability.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims AES key leakage instead of plaintext residue.
- Reports an algorithmic AES weakness unrelated to the wrapper register map.

False negative condition:

- If the input scope contains `aes0_wrapper.sv`, but the agent does not report uncleared readable plaintext registers, mark H21-039 as false negative.

## 7. Evaluation Notes

This case is related to H21-013 but narrower: H21-039 specifically concerns plaintext residue after encryption.
