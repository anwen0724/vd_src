## 2. Expected Bug Semantics

AES2 wrapper stores plaintext, internal state, and key material in memory-mapped registers. The read path exposes register contents through `rdata`, including a default pre-assignment from `key0[address[8:3]]`, while the wrapper is integrated as an externally reachable peripheral.

## 4. Hit Criteria

- Reports that AES internal/key/plaintext state is externally readable through the peripheral register interface.
- Cites `aes2_wrapper.sv` read logic around lines 153-181 or equivalent.
- Connects the wrapper to the SoC integration path.

## 5. Partial Hit Criteria

- Reports generic AES register exposure but does not identify the read path.
- Finds AES key/state registers but does not explain external readability.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims AES algorithm weakness without register exposure evidence.
- Reports debug-mode key exposure only, without tying to the memory-mapped read path.

False negative condition:

- If the input scope contains `aes2_wrapper.sv` and `riscv_peripherals.sv`, but the agent does not report externally readable AES internal registers, mark H21-013 as false negative.

## 7. Evaluation Notes

Score this as a protected crypto state exposure case, not as a cryptographic algorithm weakness.
