## 2. Expected Bug Semantics

HMAC access is gated by `acc_ctrl_c[priv_lvl_i][4]`. Since access-control entries reset to all ones, the privilege-indexed access-control matrix grants broad access unless later restricted, making HMAC protected registers reachable from user privilege.

## 4. Hit Criteria

- Reports that HMAC protected registers are accessible from user privilege due to access-control state.
- Cites both the privilege-indexed access gate and HMAC register map.
- Connects the issue to access-control default/full-access behavior rather than only HMAC internals.

## 5. Partial Hit Criteria

- Reports HMAC register exposure but omits privilege/access-control reasoning.
- Reports reset-to-full-access but does not identify HMAC as the affected target.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims HMAC cryptographic weakness without access-control evidence.
- Reports HMAC key-register locking issue only, without user-mode accessibility.

False negative condition:

- If the input scope contains `riscv_peripherals.sv`, `acct_wrapper.sv`, and `hmac_wrapper.sv`, but the agent does not report user-mode accessibility of HMAC protected registers, mark H21-074 as false negative.

## 7. Evaluation Notes

This case is related to H21-042 but target-specific: H21-042 is global reset access control; H21-074 asks whether HMAC protected registers are exposed to user-mode access.
