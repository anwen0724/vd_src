## 2. Expected Bug Semantics

The access-control memory `acct_mem` resets every entry to `32'hffffffff`. This state is exported through `acc_ctrl_o` and expanded into privilege-indexed `acc_ctrl_c`, which gates peripheral access. Reset therefore initializes access control to full access instead of least privilege.

## 4. Hit Criteria

- Reports reset initializes access-control entries to all ones/full access.
- Cites `acct_mem[j] <= 32'hffffffff`.
- Explains that `acc_ctrl_o`/`acc_ctrl_c` affects SoC peripheral access mediation.

## 5. Partial Hit Criteria

- Reports an access-control problem but not the reset-default behavior.
- Finds `acct_mem` reset but does not connect it to peripheral access gates.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports register locks reset state instead of access-control reset state.
- Claims a specific peripheral bypass without explaining the global full-access default.

False negative condition:

- If the input scope contains `acct_wrapper.sv` and `riscv_peripherals.sv`, but the agent does not report reset-to-full-access behavior, mark H21-042 as false negative.

## 7. Evaluation Notes

This is a strong permission-related case and should be kept as a primary H21 evaluation case.
