| `official_description` | CLINT registers are not protected by access control |

## 2. Expected Bug Semantics

The CLINT AXI-lite bridge is wired directly to the CLINT instance without an `acc_ctrl_c` gate. Other protected resources in the same file show explicit access-control mediation, such as ROM request gating and wrapper `acct_ctrl_i` connections. CLINT therefore lacks the intended access-control protection.

## 4. Hit Criteria

- Reports that CLINT registers lack access-control mediation.
- Cites the direct CLINT bridge wiring and contrasts it with `acc_ctrl_c`-gated resources.
- Explains this as missing access control rather than CLINT functional behavior.

## 5. Partial Hit Criteria

- Reports generic CLINT exposure without showing missing access-control gate.
- Finds `clint_axi_req` but does not compare with gated resources.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims CLINT interrupt logic is functionally wrong.
- Reports PLIC access-control issue instead of CLINT.

False negative condition:

- If the input scope contains `riscv_peripherals.sv`, but the agent does not report missing access-control mediation on CLINT registers, mark H21-099 as false negative.

## 7. Evaluation Notes

This is a strong access-control omission case and useful for evaluating whether agents compare protected and unprotected integration patterns.
