## 2. Expected Bug Semantics

The DMA wrapper receives `reglk_ctrl_i`, but core DMA programming registers such as `start_reg`, `length_reg`, source address, and destination address are written directly. Only `core_lock_reg` has a lock-like update pattern, so DMA behavior can be reprogrammed despite the intended register-lock mechanism.

## 4. Hit Criteria

- Cites direct assignments in `dma_wrapper.sv` around lines 129-146.
- Explains why source/destination/length registers are security-sensitive.

## 6. False Positive / False Negative Rules
