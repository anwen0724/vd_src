## 2. Expected Bug Semantics

`acct_wrapper` assigns an 8-bit `reglk_ctrl_i` input into local `reglk_ctrl`, but ACCT write cases reference lock bits such as `reglk_ctrl[13]`. These out-of-range or ineffective lock references mean some ACCT configuration registers are not protected by valid register-lock mediation.

## 4. Hit Criteria

- Reports that some ACCT registers are not effectively protected by register locks.
- Cites the ACCT write map and invalid/inconsistent `reglk_ctrl` indices.
- Connects ACCT registers to global access-control configuration.

## 5. Partial Hit Criteria

- Reports ACCT access-control weakness but not register-lock omission.
- Finds `reglk_ctrl[13]` but does not explain why it is ineffective.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports reset-to-full-access only.
- Reports HMAC/PKT coupling instead of ACCT register-lock protection.

False negative condition:

- If the input scope contains `acct_wrapper.sv`, but the agent does not report ineffective lock protection for ACCT registers, mark H21-043 as false negative.

## 7. Evaluation Notes

This case should be treated as register-lock coverage failure over access-control configuration registers.
