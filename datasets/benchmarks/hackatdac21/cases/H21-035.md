## 1. Case Identity

| 字段 | 内容 |
|---|---|
| `case_id` | H21-035 |
| `official_id` | #35 |
| `official_description` | Reg locks are disabled by default when reset |
| `task_category` | `reset/register_lock` |
| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

The register-lock memory `reglk_mem` is reset to zero. The resulting `reglk_ctrl_o` is exported as the global lock vector used by protected peripheral wrappers, so reset leaves register locks disabled by default.

## 4. Hit Criteria

- Reports that register locks reset to disabled/unlocked state.
- Cites `reglk_mem[j] <= 'h0` or equivalent.
- Explains that `reglk_ctrl_o` drives peripheral lock decisions.

## 5. Partial Hit Criteria

- Reports a reset issue but does not connect it to register-lock state.
- Finds `reglk_mem` reset but not its global effect.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims a reset controller bug without evidence from `reglk_wrapper.sv`.
- Reports access control reset state instead of register-lock reset state.

False negative condition:

- If the input scope contains `reglk_wrapper.sv`, but the agent does not report default-disabled register locks after reset, mark H21-035 as false negative.

## 7. Evaluation Notes

This case is close to H21-042 but distinct: H21-035 is about register-lock state, while H21-042 is about access-control state.
