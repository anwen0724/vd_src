# H21-080 Case GT

## 1. Case Identity

| 字段 | 内容 |
|---|---|
| `case_id` | H21-080 |
| `official_id` | #80 |
| `official_description` | AES1 key registers are not locked by register locks |
| `task_category` | `aes1/key_register_lock` |
| `gt_status` | `rtl_confirmed` |

## 2. Expected Bug Semantics

AES1 key registers are written under `reglk_ctrl_i[3]`, but the readback path for the same key register arrays is guarded by `reglk_ctrl_i[2]`. This inconsistent lock mapping can leave AES1 key registers insufficiently protected.

## 3. Expected RTL Evidence

| Evidence | 文件 | 行号 | 关键对象 | 评分用途 |
|---|---|---:|---|---|
| Key register definition | `third_party/hackatdac21/piton/design/chip/tile/ariane/src/aes1/aes1_wrapper.sv` | 118-122 | `key_reg0`, `key_reg1`, `key_reg2`, `key_we*` | Shows AES1 key register arrays and address hit signals. |
| Key write locking | `third_party/hackatdac21/piton/design/chip/tile/ariane/src/aes1/aes1_wrapper.sv` | 261-266 | `key_reg*`, `reglk_ctrl_i[3]` | Shows writes use lock bit 3. |
| Key read locking | `third_party/hackatdac21/piton/design/chip/tile/ariane/src/aes1/aes1_wrapper.sv` | 292-294 | `key_reg*`, `reglk_ctrl_i[2]` | Shows reads use lock bit 2. |
| AES1 integration | `third_party/hackatdac21/piton/design/chip/tile/ariane/openpiton/riscv_peripherals.sv` | 1273-1281 | `i_aes1_wrapper`, `reglk_ctrl_i` | Shows AES1 receives register-lock control. |

## 4. Hit Criteria

- Reports AES1 key register lock inconsistency.
- Cites write guard `reglk_ctrl_i[3]` and read guard `reglk_ctrl_i[2]`.
- Explains this as register-lock mediation failure over key registers.

## 5. Partial Hit Criteria

- Reports AES1 key exposure without lock-bit analysis.
- Finds key register arrays but not their inconsistent protection.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports AES0 default key leak instead.
- Claims no lock inputs exist for AES1.

False negative condition:

- If the input scope contains `aes1_wrapper.sv`, but the agent does not report inconsistent key register lock mediation, mark H21-080 as false negative.

## 7. Evaluation Notes

This case is useful for checking whether agents compare read and write protection rather than only detecting exposed signals.
