## 2. Expected Bug Semantics

AES1 masks `core_key0` and `core_key2` when `debug_mode_i` is asserted, but `core_key1` remains unmasked. Since `core_key` can select `core_key1`, key material can remain exposed to the AES core path during debug mode.

## 4. Hit Criteria

- Reports AES1 key exposure/incomplete masking in debug mode.
- Cites unmasked `core_key1` while `core_key0` and `core_key2` are masked.
- Explains how `core_key` selection can carry the unmasked key.

## 5. Partial Hit Criteria

- Reports debug-mode AES issue but does not identify AES1/core_key1.
- Finds key registers but not debug masking inconsistency.

## 6. False Positive / False Negative Rules

False positive examples:

- Reports AES0 key2 debug issue only.
- Reports AES register exposure unrelated to debug mode.

False negative condition:

- If the input scope contains `aes1_wrapper.sv`, but the agent does not report incomplete debug key masking, mark H21-098 as false negative.

## 7. Evaluation Notes

This case is distinct from H21-047 because it targets AES1 `core_key1` debug masking.
