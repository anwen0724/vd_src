## 2. Expected Bug Semantics

The HMAC reset block clears control and data registers, but does not clear key-related registers such as `key0`, `ikey_hash_bytes`, and `okey_hash_bytes`. These sensitive registers can therefore retain state across reset paths.

## 4. Hit Criteria

- Reports HMAC key/key-derived registers are not reset.
- Cites reset block and omitted key registers.
- Explains residual sensitive state after reset.

## 5. Partial Hit Criteria

- Reports HMAC key exposure but not reset residue.
- Finds reset block but does not identify omitted sensitive registers.

## 6. False Positive / False Negative Rules

False positive examples:

- Claims SHA reset issue.
- Reports HMAC register-lock inconsistency only.

False negative condition:

- If the input scope contains `hmac_wrapper.sv`, but the agent does not report key-related HMAC registers missing from reset, mark H21-097 as false negative.
